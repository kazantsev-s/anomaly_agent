# Агент по поиску аномалий

## Предметная область

Данные были собраны с платформы Kolesa.kz - крупнейшная Казахстанская онлайн площадка автообъявлений по продаже авто. Основная доля объявлений - частные, чаще б/у. Количество объявлений достаточно большое, платформе важно, чтобы они были настоящими и корректно заполненными, чтобы пользователи в ленте видели как можно больше качественных объявлений, что повысило бы статус площадки, а продавцу важно как можно скорее найти покупателя.

В связи с этим агент будет искать среди опубликованных объявлений подозрительные, которые могут свидетельствовать о мошенничестве или некорректном заполнении данных.

## Реализация проекта

Проекта представляет из себя AI-агента, который имеет доступ к данным, проводить полный анализ по поиску аномалий и умеет обрабатывать отдельные запросы пользователя. Для ответа на его вопросы и понимания представленных данных - агент умеет формировать и выполнять SQL запросы к базе данных, как заранее заготовленные, так и полностью кастомные, что является обязательным этапом при обработке запроса.

В проекте есть два режима работы:

- ответ на пользовательский вопрос через `/ask`;
- полный анализ аномалий таблицы через `/analyze`.

### Взаимодействие с агентом

Общение с агентом выстраивается через Telegram-бота, в котором работают следующие команды:

- `/start` - показывает краткое приветствие и подсказывает проверить подключение к базе.

- `/health` - проверяет доступность БД.

- `/ask {запрос}` - описание запроса для AI-агента (`/ask` и после пробела любой текстовый запрос).

- `/analyze` - запускает полный анализ таблицы: агент собирает схему и профиль данных, выполняет базовые проверки аномалий, при необходимости запускает дополнительные SQL-проверки и возвращает итоговый отчет.

### Демонстрационный агент

Tелеграм бот, через которого реализуется взаимодействие с демонстрационным агентом (требует запущенных контейнеров на стороне хоста):

[@anomalyagent_bot](t.me/anomalyagent_bot)

### Используемый технологический стек

Проект написан на `Python`. В качестве СУБД используется `PostgreSQL`, запуск приложения и базы данных организован через `Docker` и `Docker Compose`.

Для работы с PostgreSQL используется асинхронная библиотека `asyncpg`: она отвечает за подключение к базе данных, инициализацию таблицы, загрузку данных из CSV и выполнение SQL-запросов.

Telegram-бот реализован с помощью библиотеки `aiogram`. Бот обрабатывает команды пользователя, проверяет подключение к базе данных и передаёт запросы AI-агенту.

Взаимодействие с LLM реализовано через официальный клиент `openai`, в частности через асинхронный клиент `AsyncOpenAI` и `Responses API`.

Логика AI-агента построена с использованием `LangGraph`. В проекте есть два графа:

- `agent_graph` - простой граф для команды `/ask`: генерация SQL-запроса, выполнение SQL и подготовка ответа пользователю.
- `anomaly_agent_graph` - граф полного анализа для команды `/analyze`: загрузка схемы, профилирование таблицы, запуск проверок, дополнительные SQL-проверки и финальный отчет.

Для загрузки переменных окружения используется `python-dotenv`.

### Ответ на запрос `/analyze`

В бот была просто отправлена команда `/analyze`, на которую агент сформировал ответ:

```text

Проверено 5396 строк таблицы kolesa. Основные реальные проблемы — несогласованность полей по изображениям. Большинство ценовых, объемных и возрастных выбросов выглядят как валидные редкие случаи рынка: дорогие премиальные авто, крупнообъемные SUV/AMG и ретро-авто.

2. Основные проблемы

Реальные ошибки данных
- found_img = false, но img_url заполнен: 80 объявлений.
- imgs_count > 0, но found_img = false: 76 объявлений.
- imgs_count = 0, но found_img = true: 68 объявлений.

Примеры:
- id 252, kolesa_id 222609623, Audi A4 1996: объявление
- id 287, kolesa_id 212133908, Audi TT 2000: объявление
- id 29, kolesa_id 219403477, Chevrolet Tahoe 2026, 0 фото, но фото найдено: объявление

Вероятно валидные редкие случаи
- 22 объявления с engine_volume = 0 относятся к электромобилям. Это может быть нормальным правилом кодирования для электромобилей.
- 431 объявление с объемом двигателя вне IQR в основном крупнообъемные модели: Lexus LX 570, Lexus LX 470, Toyota Land Cruiser, Mercedes-Benz S/G/GL AMG. Это похоже на валидные значения.
- 441 объявление с ценой выше IQR-порога 27 600 000 тг сформированы дорогими моделями BMW X5/X7, Lexus LX/GX, Mercedes-Benz G 63 AMG, Toyota Land Cruiser.
- 4 объявления с body_type Родстер: BMW Z4 и Mercedes-Benz SL 500. Это похоже на валидную редкую категорию, а не на ошибку.

3. Детальные находки

Изображения
Custom SQL показал:
- 190 объявлений имеют высокое количество фото: от 40 до 99 фото, при этом found_img = true и ссылки на изображения заполнены. Это скорее редкие, но валидные объявления.
- 74 объявления: количество фото от 2 до 35, found_img = false, но img_url и img_filename заполнены — явная несогласованность.
- 68 объявлений: imgs_count = 0, но found_img = true и есть URL/filename изображения — явная несогласованность.
- 20 объявлений: imgs_count = 0, found_img = false, URL и filename пустые — выглядит согласованно.

Пробег
IQR нашел 70 объявлений с пробегом выше 501 917 км. Custom SQL показывает повторяющиеся «круглые/шаблонные» значения:
- 777777 км встречается в 10 объявлениях;
- 555555 км встречается в 8 объявлениях;
- 550000 км встречается в 6 объявлениях.

Пример подозрительного экстремума: id 269, kolesa_id 220658975, Audi A6 1995, пробег 999999 км: объявление

Год выпуска
Есть 3 объявления старше IQR-порога:
- Mercedes-Benz E 230 1977, цена 12 000 000 тг;
- BMW 316 1983, цена 10 000 000 тг;
- BMW 732 1983, цена 2 400 000 тг.
По моделям это похоже на ретро/редкие авто, не обязательно ошибка.

4. Что проверить вручную

- Логику расчета found_img: почему флаг false при заполненных img_url и img_filename.
- Источник imgs_count: почему бывает 0 фото при найденном изображении.
- Повторяющиеся пробеги 777777 км, 555555 км, 999999 км — это могут быть заглушки продавцов или парсинга.
- Не считать engine_volume = 0 ошибкой без проверки fuel_type: для электромобилей это может быть нормальным кодированием.

5. Итоговая оценка качества данных

Качество данных среднее. Критичных массовых ошибок по основным атрибутам авто не видно, но блок изображений требует исправления правил заполнения. Цены, объемы двигателя, старые годы выпуска и редкий body_type в основном объясняются реальными особенностями объявлений.
```

### Ответ на запрос `/ask`

В данном кейсе пользователем был задан конкретный вопрос, на который агент ответил сформировав один SQL запрос (далее сделать чтобы мог делать несколько)

В бот было отправлено сообщение:

```text
/ask Есть ли в данных подозрительные объявления в соотношении цены и года автомобиля. То есть условно новые автомобили по низкой цене
```

Был получен следующий ответ:

```text
Найдено **1 подозрительное объявление** по критерию “новый/свежий автомобиль с ценой сильно ниже рынка”.

### Подозрительное объявление

| ID | Авто | Год | Пробег | Город | Цена | Медиана по таким же авто | Отклонение |
|---|---|---:|---:|---|---:|---:|---:|
| [222075136](https://kolesa.kz/a/show/222075136?search_id=9e003aff8436d9a909e2489c1c9325de33ecfdfa&search_type=filter&advert_position=535) | Hyundai Elantra | 2023 | 140 000 км | Алматы | 2 300 000 ₸ | 8 400 000 ₸ | на 72.6% ниже медианы |

### Почему это выглядит аномально

Для **Hyundai Elantra 2023 года** в данных было найдено **17 объявлений**, то есть сравнение не по одному-двум случайным авто, а по небольшой, но уже более-менее релевантной группе.

Медианная цена по Hyundai Elantra 2023 года — около **8.4 млн ₸**, средняя — около **8.1 млн ₸**. А это объявление стоит всего **2.3 млн ₸**, то есть почти в **3.7 раза дешевле медианы**.

Для автомобиля 2023 года такая цена выглядит крайне низкой.

### Что может объяснять низкую цену

Объявление не обязательно мошенническое, но его стоит проверить особенно внимательно. Возможные причины:

1. **Очень большой пробег**  
   Указано **140 000 км** для авто 2023 года — это много для почти нового автомобиля. Возможно, машина работала в такси/коммерческом использовании. Это снижает цену, но падение на 72.6% всё равно выглядит слишком сильным.

2. **Серьёзные повреждения или восстановление после ДТП**  
   Цена может быть такой, если авто аварийное, после сильного ремонта, с проблемами по кузову/геометрии.

3. **Не полная цена в объявлении**  
   Иногда в цене указывают первоначальный взнос, остаток кредита/лизинга отдельно, либо цена “от”.

4. **Ошибка в объявлении**  
   Возможна опечатка в цене, годе выпуска или модели.

5. **Юридические ограничения**  
   Залог, арест, кредит, проблемы с документами.

6. **Потенциально мошенническое объявление**  
   Сильно заниженная цена на свежий автомобиль может использоваться для привлечения внимания.

### На что обратить внимание

В первую очередь стоит проверить именно это объявление:

- **Hyundai Elantra 2023, ID 222075136**
- цена: **2.3 млн ₸**
- медиана по аналогам: **8.4 млн ₸**
- пробег: **140 тыс. км**
- город: **Алматы**

Рекомендуется проверить описание объявления, фото, VIN/госномер, историю ДТП, юридическую чистоту, а также уточнить у продавца, является ли цена полной или это взнос/часть стоимости.

Итог: по выполненному запросу найдено **одно явно выбивающееся объявление** — свежий Hyundai Elantra 2023 года с ценой существенно ниже рынка.
```

## Локальный запуск

Запуск происходит через Docker Compose. С отдельным поднятием БД и остальными сервисами.

### Подготовка

1. Скопировать `.env.example` в `.env`:

    *Mac/Linux:*
    
    ```bash
    cp .env.example .env
    ```

    *Windows:*
  
    ```bash
    copy .env.example .env
    ```

2. Вставить реальный `TG_TOKEN` в `.env`.

3. Вставьте OpenAI API ключ для взаимодейсвтия с LLM от OpenAI (GPT)

4. Указать пароль для БД в `POSTGRES_PASSWORD`

### Запуск контейнеров

Далее этапы запуска через docker compose контейнеров отдельно для базы данных PostreSQL и остальных сервисов (Агент, Бот)

### Запуск PostgreSQL

1. Запуск контейнера PostgreSQL:

    ```bash
    docker compose -f docker-compose.db.yaml up -d
    ```

2. Посмотреть логи базы:

    ```bash
    docker compose -f docker-compose.db.yaml logs -f db
    ```

3. Проверить статус:

    ```bash
    docker compose -f docker-compose.db.yaml ps
    ```

4. Подключиться к базе данных:

    Используйте значения `.env`
    По умолчанию:
   
      - Host: `localhost`
      - Port: 5432
      - Database: `kolesa`
      - User: `kolesa`
      - Password: `установленный_в_.env`

6. Остановить контейнер с БД

    Обычная остановка:
    
    ```bash
    docker compose -f docker-compose.db.yaml down
    ```

    Удалить базу вместе с данными:
    
    ```bash
    docker compose -f docker-compose.db.yaml down -v
    ```

### Запуск остальных сервисов

Может выполняться независимо от запуска БД

1. Запуск контейнера с выводом логов в терминал:

    Обычный запуск

    ```bash
    docker compose up --build
    ```

    Фоновый запуск:

    ```bash
    docker compose up --build -d
    ```

2. Посмотреть логи:

    ```bash
    docker compose logs -f
    ```

3. Проверить статус:

    ```bash
    docker compose ps
    ```

4. Остановить контейнер:

    ```bash
    docker compose down
    ```
